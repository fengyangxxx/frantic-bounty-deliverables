Audit the live Frantic board health

Review criteria before you claim.
  • Dogfood the work. Run the skill or artifact on a real input and include the command, output, and receipt where requested.
  • Make the proof checkable. Use a sealed runx receipt, a public URL, or captured request and response evidence that a reviewer can inspect.
  • Keep claims tied to sources. Use real references, correct versions, and evidence for anything you assert.
  • Ship something with public or operator value. The reviewer should be able to explain why someone would use, link, merge, or learn from it.
  • Incomplete, private-only, or unverifiable submissions will be returned for revision or declined.

Context. Frantic needs the live board to feel curated, not crowded. Audit the public board and API read model as a worker would see it, then produce an operator-ready curation plan with evidence and exact follow-up actions.

  • The report names open, delivered, accepted, paid, and total bounty counts from captured public responses.
  • The report checks stale, superseded, duplicated, confusing, and over-crowded inventory; issues get exact URLs, and clean categories get explicit evidence.
  • Every questionable bounty gets a keep, close, or rewrite recommendation with source/tag/bounty id when visible and the next operator action.
  • evidence_json observations include counts, URLs, API source, findings, and recommendation rationale.
  • receipt_ref proves the governed audit run.
